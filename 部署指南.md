# GitHub Pages 部署指南

## 📋 概述

将 FitCoach AI 应用部署到 GitHub Pages，实现在任何设备上通过浏览器访问。

---

## 🔧 步骤一：创建 GitHub 仓库

1. 访问 https://github.com/new
2. 填写仓库信息：
   - **Repository name**: `fitcoach-ai` (或其他你喜欢的名字)
   - **Description**: `FitCoach AI - 私人运动教练应用`
   - **Public/Private**: 选择 **Public** (GitHub Pages 免费版只支持公开仓库)
   - **Don't initialize** (保持其他选项不选)
3. 点击 **Create repository**

---

## 🔧 步骤二：关联本地仓库

将下面命令中的 `YOUR_USERNAME` 替换为你的 GitHub 用户名，然后在当前工作目录运行：

```bash
# 添加远程仓库（请替换 YOUR_USERNAME）
git remote add origin https://github.com/YOUR_USERNAME/fitcoach-ai.git

# 推送到 GitHub
git branch -M main
git push -u origin main
```

**示例**（如果你的用户名是 `zhangming`）：
```bash
git remote add origin https://github.com/zhangming/fitcoach-ai.git
git branch -M main
git push -u origin main
```

---

## 🔧 步骤三：启用 GitHub Pages

1. 打开你刚创建的 GitHub 仓库页面
2. 点击顶部的 **Settings** 标签
3. 在左侧菜单找到 **Pages**
4. 配置如下：
   - **Source**: 选择 `Deploy from a branch`
   - **Branch**: 选择 `main` 分支，文件夹选择 `/(root)`
   - 点击 **Save**

5. 等待 1-2 分钟，页面会显示部署状态：
   ```
   Your site is live at: https://YOUR_USERNAME.github.io/fitcoach-ai/
   ```

---

## ✅ 完成访问

部署完成后，你可以在任何设备的浏览器中访问：

```
https://YOUR_USERNAME.github.io/fitcoach-ai/
```

## 🔄 如何更新应用

当你修改了代码后，运行以下命令推送更新：

```bash
git add ai-coach/
git commit -m "更新描述"
git push
```

GitHub Pages 会在 1-2 分钟内自动重新部署。

---

## 🛠️ 可选：自定义域名

如果你有自己的域名，可以在 GitHub Pages 设置中添加：

1. 在 **Pages** 设置中找到 **Custom domain**
2. 输入你的域名（例如：`fitcoach.example.com`）
3. 在你的域名 DNS 设置中添加 CNAME 记录：
   ```
   CNAME    fitcoach    YOUR_USERNAME.github.io
   ```

## 💡 提示

- GitHub Pages 免费版支持：
  - 无限带宽
  - 自动 HTTPS (SSL 证书)
  - 自定义域名支持
  - 持续集成自动部署
- 网页是静态的，刷新页面数据会重置（需要本地存储或后端服务来持久化数据）

---

## 🆘 常见问题

**Q: 部署后页面显示 404？**
A: 检查 GitHub Pages 的 Branch 设置是否为 `main/(root)`

**Q: 修改代码后没有更新？**
A: 等待 1-2 分钟，或者清理浏览器缓存（Ctrl + F5）

**Q: 能否添加后端服务？**
A: GitHub Pages 只支持静态网站。需要后端可以改用 Vercel、Netlify 或自己的服务器
