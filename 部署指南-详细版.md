将 FitCoach AI 应用部署到 GitHub Pages，实现在任何设备上通过浏览器访问。

## 🔧 步骤一：创建 GitHub 仓库

1. 访问 https://github.com/new
2. 填写仓库信息：
   - **Repository name**: `fitcoach-ai`
   - **Description**: `FitCoach AI - 私人运动教练应用`
   - **Public/Private**: 选择 **Public** (GitHub Pages 免费版只支持公开仓库)
   - **Add .gitignore**: 不选
   - **Choose a license**: 不选
3. 点击 **Create repository**

---

## 🔧 步骤二：推送代码到 GitHub

仓库创建成功后，GitHub 会显示推送命令。请在 PowerShell 中运行：

```bash
cd c:/Users/yezhenzhen/WorkBuddy/20260318141233
git push -u origin main
```

如果提示需要登录，请按照提示输入你的 GitHub 用户名和 **Personal Access Token**。

**⚠️ 重要提示**：GitHub 已停止使用密码验证，需要使用 Token：
1. 访问 https://github.com/settings/tokens
2. 点击 **Generate new token (classic)**
3. 勾选 `repo` 权限
4. 生成 Token 并复制
5. 推送时输入 Token（不会显示字符，这是正常的）

---

## 🔧 步骤三：启用 GitHub Pages（新版本）

仓库推送成功后：

1. 打开你的仓库页面：https://github.com/merx6/fitcoach-ai
2. 点击顶部的 **Settings** 标签
3. 在左侧菜单中找到 **Pages**
4. 点击 **Pages**

### 配置选项

GitHub Pages 有两种模式，根据你看到的界面选择：

---

### 模式 A：GitHub Actions（推荐）

如果看到以下界面：

> **Build and deployment** → **Source** 选择 **GitHub Actions**

1. 点击 **Source** 下拉菜单
2. 选择 **GitHub Actions**
3. 会看到一个静态网站的工作流配置模板
4. 点击 **Commit changes...** 或直接使用默认配置

GitHub 会自动创建一个 GitHub Actions 工作流来部署你的站点。

---

### 模式 B：Deploy from a branch（传统模式）

如果看到以下界面：

> **Source** 选择 **Deploy from a branch**

1. 点击 **Source** 下拉菜单
2. 选择 **Deploy from a branch**
3. **Branch** 选择 `main` 分支
4. **Folder** 选择 `/ (root)` （根目录）
5. 点击 **Save**

---

## 📋 关于 "/(root)" 选项

**旧版界面**：
```
Branch: main
Folder: / (root)  ← 有这个选项
```

**新版界面**：
```
Branch: main
Directory: / (root) ← 可能叫 "Directory" 或 "Folder"
```

如果仍然看不到根目录选项，请检查：

1. 确认选择了 `main` 分支
2. 点击文件夹下拉菜单，应该会有 `/ (root)` 和 `/docs` 两个选项
3. 选择 `/ (root)`

---

## 🕐 等待部署

1. 在 **Pages** 页面查看部署状态
2. 通常需要 **1-3 分钟**完成
3. 完成后会显示：


## 🛠️ 常见问题

### Q1: 看不到 "GitHub Actions" 选项？

**A**: 新的 GitHub Pages 可能只显示 "Deploy from a branch"，这是正常的。选择这个模式也可以。

### Q2: 看不到 "/ (root)" 文件夹选项？

**A**: 
- 先选择 `main` 分支
- 然后点击 **Folder** 或 **Directory** 下拉菜单
- 应该会显示 `/ (root)` 和 `/docs` 两个选项
- 选择 `/ (root)`

### Q3: 推送时提示 "Repository not found"？

**A**: 
- 确认仓库地址正确：https://github.com/merx6/fitcoach-ai
- 确认仓库已创建
- 确认使用的是你的 GitHub 账号登录

### Q4: 推送时要求密码？

**A**: 
- GitHub 已不再支持密码验证
- 需要使用 Personal Access Token
- 访问 https://github.com/settings/tokens 创建 Token
- Token 权限选择 `repo`

### Q5: 部署后页面显示 404？

**A**: 
- 检查 GitHub Pages 的 Branch 设置是否为 `main`
- 检查 Folder 是否为 `/(root)`
- 等待 2-3 分钟让部署完成

### Q6: 如何修改文件后重新部署？

**A**: 
```bash
git add ai-coach/
git commit -m "更新描述"
git push
```
GitHub Pages 会自动重新部署。
