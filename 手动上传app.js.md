# 手动上传 app.js 到 GitHub（详细步骤）

## 📋 目标
将更新后的 `app.js` 上传到 GitHub 仓库 `fitcoach-ai-`，启用本地存储功能。

---

## 🚀 上传步骤（图文指引）

### 第 1 步：打开仓库页面
访问：https://github.com/merx6/fitcoach-ai-

---

### 第 2 步：找到并删除旧文件

1. 点击仓库中的 **ai-coach** 文件夹
2. 点击 **app.js** 文件
3. 在文件内容页面右上角，点击 **...** 按钮（三个点）
4. 在下拉菜单中选择 **Delete file**
5. 页面会跳转到删除确认页面

---

### 第 3 步：确认删除

在删除确认页面：
- **Commit message** 会自动填写 `Delete app.js`
- 确认无误后，点击绿色的 **Commit changes** 按钮
- 等待页面刷新，文件列表中应该不再有 `app.js`

---

### 第 4 步：上传新文件

1. 回到仓库主页：https://github.com/merx6/fitcoach-ai-
2. 点击绿色的 **Add file** 按钮
3. 在下拉菜单中选择 **Upload files**

---

### 第 5 步：选择文件

在上传页面：

1. 点击大的 **choose your files** 按钮
2. 在文件选择窗口中，导航到这个路径：
   ```
   C:\Users\yezhenzhen\WorkBuddy\20260318141233\ai-coach\
   ```
3. 选择 **app.js** 文件
4. 文件会被添加到上传列表

---

### 第 6 步：填写提交信息

在页面右侧或底部的提交信息区域：

**Commit message** 输入：
```
feat: 添加本地存储功能，保存健康档案数据
```

---

### 第 7 步：提交更改

点击绿色的 **Commit changes** 按钮

---

### 第 8 步：等待自动部署

上传成功后，GitHub 会自动触发 Pages 部署：
1. 等待 **1-2 分钟**
2. 访问：https://github.com/merx6/fitcoach-ai-/settings/pages
3. 查看部署状态，应该显示绿色的对勾 ✓

---

## ✅ 验证功能

部署完成后，测试本地存储功能：

### 测试步骤

1. **打开应用**
   访问：https://merx6.github.io/fitcoach-ai-/

2. **修改数据**
   - 点击侧边栏的"健康档案"
   - 修改姓名（例如改为"测试用户"）
   - 修改年龄、体重等信息
   - 点击底部的"保存档案"按钮

3. **验证保存成功**
   - 应该看到提示：`✓ 健康档案已保存，AI 已重新分析训练计划`
   - 控制台（F12）会显示：`数据已保存到本地存储`

4. **刷新页面**
   - 按 **F5** 键刷新页面
   - 检查姓名是否仍显示"测试用户"
   - 检查其他修改的信息是否保留

5. **关闭浏览器重新打开**
   - 完全关闭浏览器窗口
   - 重新打开浏览器访问应用
   - 数据应该仍然保留

---

## 🎯 功能说明

### 本地存储做什么？
- 保存位置：浏览器的 LocalStorage
- 保存内容：完整的用户健康档案数据
- 保存时机：每次点击"保存档案"按钮
- 加载时机：每次打开或刷新页面

### 数据会保留在哪里？
| 场景 | 是否保留 |
|------|---------|
| 刷新页面 | ✅ 保留 |
| 关闭浏览器再打开 | ✅ 保留 |
| 同一设备不同标签页 | ✅ 保留 |
| 清除浏览器缓存 | ❌ 丢失 |
| 换个设备访问 | ❌ 丢失 |
| 换个浏览器访问 | ❌ 丢失 |

---

## 💡 提示

- 如果上传失败，可以多试几次
- 如果文件很大，上传可能需要几秒钟
- 上传完成后，GitHub 会显示成功的绿色对勾
- Pages 部署需要 1-2 分钟，请耐心等待

---

## 🆘 问题排查

**Q: 上传后页面还是旧版本？**
A: 清除浏览器缓存（Ctrl + F5），等待 2 分钟让 Pages 部署完成

**Q: 点击"保存档案"后刷新，数据没有保留？**
A: 打开控制台（F12）查看是否有错误信息

**Q: 看不到上传界面？**
A: 确保你登录了 GitHub 账号
