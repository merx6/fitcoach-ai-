## 🎯 功能说明

### 保存机制
- 点击"保存档案"按钮时，自动保存到 `localStorage`
- 存储键名：`fitcoach_data`
- 存储内容：完整的 `UserProfile` 对象

### 加载机制
- 页面加载时（DOMContentLoaded）自动从 `localStorage` 读取
- 如果有保存的数据，自动更新 `UserProfile` 对象
- 如果没有保存的数据，使用默认值

### 数据范围
保存的数据包括：
- 基本信息（姓名、年龄、性别、身高、体重等）
- 健康参数（静息心率、最大心率、VO₂Max 等）
- 运动偏好（训练目标、每周训练天数、训练时长等）
- 健康注意事项

## 🔧 技术细节

### saveToLocalStorage()
```javascript
function saveToLocalStorage() {
  const data = {
    userProfile: UserProfile,
    timestamp: Date.now()
  };
  localStorage.setItem('fitcoach_data', JSON.stringify(data));
}
```

### loadFromLocalStorage()
```javascript
function loadFromLocalStorage() {
  const saved = localStorage.getItem('fitcoach_data');
  if (saved) {
    const data = JSON.parse(saved);
    Object.assign(UserProfile, data.userProfile);
    return true;
  }
  return false;
}
```

## 📱 局限性说明

**本地存储的局限性：**
- ✅ 同一浏览器、同一台设备上数据会保留
- ❌ 不同设备之间不会同步
- ❌ 不同浏览器之间不会同步（Chrome 和 Edge 各自独立）
- ❌ 清除浏览器缓存会丢失数据

**如果需要跨设备同步，可以：**
1. 使用 Supabase（免费，需要配置）
2. 使用 Firebase（免费，需要配置）
3. 搭建自己的后端服务
